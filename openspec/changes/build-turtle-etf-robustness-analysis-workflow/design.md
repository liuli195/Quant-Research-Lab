## Context

前置变更 `build-turtle-etf-joinquant-backtest-workflow` 负责在 JoinQuant（聚宽）云端生成会改变交易路径的正式基线和稳定性场景，并把明确回测对象归档到仓库。当前仓库已有聚宽归档结构和完整性门禁，但尚无统一入口把多个正式快照组织为可复现的稳健性、压力和尾部风险分析，也没有逐项记录来源、阈值和判定的证据矩阵。

本变更同时服务两类使用者：需要复用量化分析流程的 Agent（代理）或研究者，以及需要按照已确认方案评估海龟 ETF（交易型开放式指数基金）的项目成员。仓库规则要求正式交易路径只来自聚宽；因此本地分析只能读取完整、可追溯的正式快照，不能补算成交、重建持仓或把本地结果声明为新的正式回测。

另一个关键约束是复用边界：`analyze-quant-robustness` Skill（技能）只负责编排，重复且易漂移的计算沉淀为确定性脚本。海龟资产、参数、场景、压力定义和门槛必须由项目配置传入，并存放在 Skill 目录之外。

## Goals / Non-Goals

**Goals:**

- 建立与具体策略解耦、配置驱动的稳健性分析流程，接收明确的正式结果清单和分析配置。
- 以确定性脚本统一计算收益与回撤指标、Block Bootstrap（区块自助抽样）、历史压力、配置化假设冲击、CVaR（条件风险价值）和硬门槛证据矩阵。
- 保留输入文件摘要、计算配置、随机种子、脚本版本和输出之间的可追溯关系，使同一输入能够复现同一结果。
- 使用通用流程完成海龟 ETF 的基线、参数、时期、资产、成本执行、风险路径和尾部风险复核，并输出唯一的“通过”“不通过”或“证据不足”结论。
- 通过结构校验、确定性测试、发布入口 E2E（端到端）回归和非海龟策略前向验证，证明 Skill 既能工作又未耦合海龟系统。

**Non-Goals:**

- 不在本地运行或重建正式交易路径，不补算缺失订单、成交或持仓。
- 不修改聚宽正式快照、策略规则、参数、资产池或验收门槛，不按结果选择较优参数或删除不利场景。
- 不冻结代码、参数或证据，不启动前向模拟或实盘，也不增加新的运行时风控规则。
- 不在 Skill（技能）目录保存海龟策略代码、业务数据、正式结果或项目专属配置。
- 不重复实现聚宽认证、回测运行或归档同步能力。

## Decisions

### 1. 采用“薄 Skill 编排 + 确定性脚本 + 项目配置”三层结构

`.agents/skills/analyze-quant-robustness/SKILL.md` 只描述触发条件、前置检查、调用顺序、停止状态和结果交付。通用计算放入同一 Skill 的 `scripts/`，输入约定和格式说明放入 `references/`；海龟项目的正式结果清单、分析配置和报告保存在业务目录，通过命令参数传入。

数据流固定为：

```text
明确正式快照清单 + 项目分析配置
→ 输入完整性与摘要校验
→ 通用确定性分析脚本
→ 逐项证据矩阵
→ 通过 / 不通过 / 证据不足
```

选择该结构是因为 Skill 适合稳定流程编排，而精确数值计算需要可独立测试的低自由度脚本；配置层则隔离不同策略的场景与门槛。

替代方案：把计算公式直接写进 `SKILL.md`。该方案依赖 Agent 临场重写计算，难以保证数值复现，故不采用。另一替代方案是建立海龟专用分析程序；它更快完成单次任务，但会把资产、参数和门槛固化进通用能力，故不采用。

### 2. 只接受显式、不可变且已通过门禁的正式证据

分析清单必须逐项列出正式回测身份、场景角色、归档路径、来源链接、代码与参数摘要、所需数据集及文件 SHA256（完整性摘要）。输入加载器先验证对象完整性门禁、文件存在性、摘要和跨场景口径；任何必需证据缺失、损坏、身份冲突或口径不一致时，流程保留已发现问题并输出“证据不足”，不得自动选择 `latest`（最新）、扫描替代快照或构造缺失交易数据。

选择显式清单可以防止归档新增回测后分析对象静默漂移，并能把每一项结论绑定到唯一来源。

替代方案：按策略目录自动寻找最新完整回测。该方案省去清单维护，但无法证明使用的是预先确认对象，也会破坏重复执行，故不采用。

### 3. 分析核心使用纯计算接口和版本化配置

通用脚本分离输入读取、指标计算、区块抽样、历史窗口、持仓冲击、CVaR 和证据矩阵聚合。计算函数只接收规范化表格或数组及显式配置，不读取海龟目录或方案文档；所有影响结果的年化口径、区块长度、路径长度、路径数量、随机种子、压力窗口、冲击映射、尾部定义和门槛都必须进入可摘要的配置。

Block Bootstrap（区块自助抽样）使用连续收益区块和固定随机种子；大样本按批次计算，以控制内存占用，但批次大小不得改变结果。Historical CVaR（历史条件风险价值）保留全部极端收益并把分位点相同收益纳入尾部样本。持仓冲击只对聚宽快照中已存在的每日收盘后真实持仓做静态重估，不产生后续订单或新的交易路径。

选择纯计算接口便于用小型固定样例验证公式，并让非海龟策略复用同一实现。

替代方案：直接在报告脚本中串联读取、计算和渲染。该方案文件较少，但输入错误、数值错误和展示错误无法独立定位，故不采用。

### 4. 使用逐项三态证据矩阵形成整体结论

每个硬门槛记录稳定要求编号、输入快照、配置与脚本摘要、计算值、比较符号、门槛和 `通过`、`不通过` 或 `证据不足` 判定。聚合顺序固定为：存在任何必需证据缺失或不可验证项时，整体为“证据不足”并同时保留已知失败；证据完整后，只要一项硬门槛失败，整体为“不通过”；仅全部硬门槛通过时才为“通过”。比较基准或较高收益不能抵消硬门槛失败。

选择三态模型可区分“已经失败”和“还无法裁决”，避免把数据缺口误报为通过或失败。

替代方案：把缺失值统一当失败。该方案虽然保守，却掩盖需要补充证据而非修改策略的真实原因，故不采用。

### 5. 海龟实例只由项目配置表达已确认方案

海龟配置列出完整正式基线、六个参数邻域变体、三个固定时期和季度移动的三年滚动窗口、11 个逐 ETF 删除变体、6 个逐资产组删除变体、5 个成本与执行压力场景，以及基于正式基线的区块抽样、固定历史压力、四个假设冲击和 CVaR。门槛、场景分组和不可补偿规则均在项目配置中声明，通用代码只解释通用格式。

选择项目配置作为唯一专属入口，可以让方案审阅者直接复核海龟要求，同时保持 Skill 和脚本在删除海龟项目文件后仍能通过通用测试。

替代方案：在通用脚本中提供 `turtle-etf` 预设。该方案调用简短，但会形成隐式专用分支并使后续门槛变更污染共用能力，故不采用。

### 6. 发布入口和解耦同时作为验收对象

验证分为五层：确定性函数单元测试、配置与输入契约测试、Skill Creator 的 `quick_validate.py`（快速校验）、仓库 Skill 与 `.claude/skills/` 兼容入口的布局校验、从发布入口运行的完整 E2E（端到端）回归。E2E 使用临时目录和小型固定正式快照样例，覆盖输入门禁、全部分析器、证据矩阵和三态结论；单元测试集合不能替代该流程。

此外，使用不包含 ETF 或海龟字段的非海龟最小策略样例，由独立子 Agent（子代理）仅根据 Skill 和原始输入执行一次前向验证。验证过程不得接触真实远端系统；海龟项目文件移除后，通用结构校验和测试仍必须通过。

选择多层验证是为了分别证明格式正确、数值正确、发布入口可用和能力真正通用。

替代方案：只运行单元测试。该方案无法发现 Skill 路由、路径布局和实际编排断裂，故不采用。

## Risks / Trade-offs

- [正式归档缺少收益、持仓、共同止损或 N 值等必需字段] → 在分析前执行字段级门禁并输出“证据不足”；只有聚宽回测流程补齐对应正式证据后才能重跑。
- [不同回测场景的日期、费用、代码或参数口径不一致] → 在清单中声明允许变化字段并对其他摘要做跨场景校验，发现未声明差异即停止裁决。
- [数值库、年化或尾部定义变化导致结果漂移] → 版本化公式约定与运行环境，保存配置和脚本摘要，并以固定黄金样例验证边界值。
- [30,000 条三年区块路径占用较多时间和内存] → 使用确定性批处理并只保留门槛所需的聚合值与可审计摘要；不得通过减少路径或删除亏损区块提速。
- [通用配置格式增加首次接入成本] → 提供最小非海龟样例和严格错误信息；接受少量配置成本以换取复现性和解耦。
- [静态假设冲击被误读为正式回测] → 报告明确标注其为对正式持仓的本地重估，不生成订单、成交或新的聚宽运行身份。
- [Windows 下兼容 Skill 符号链接失效] → 布局测试同时验证真实目录唯一性、相对链接目标和两入口内容摘要。

## Migration Plan

1. 以新增方式建立唯一真实仓库 Skill、通用脚本、输入配置约定和测试，不改动现有聚宽归档能力与历史快照。
2. 建立 `.claude/skills/analyze-quant-robustness` 到真实 Skill 目录的仓库相对兼容入口，并先通过 `quick_validate.py`、布局校验、单元测试、E2E 回归和非海龟前向验证。
3. 在聚宽回测流程交付明确且完整的正式快照后，添加海龟项目清单和配置，生成证据矩阵与整体结论；若证据不完整，则保留“证据不足”报告等待上游补齐。
4. 本变更不迁移、覆盖或删除任何正式结果。需要回滚时只移除新增 Skill、兼容入口、通用分析文件和本地分析产物，原始聚宽归档保持不变。

## Open Questions

- 海龟正式策略编号、各场景回测编号和文件摘要要等前置聚宽回测流程完成后写入分析清单，当前不能预填或使用“最新回测”替代。
- 最终运行前需要在项目配置中明确固定随机种子的具体数值；该数值必须在查看 Bootstrap（区块自助抽样）结果前确定并归档。
- 最终运行前需要固定 CAGR（复合年增长率）的精确年化日历口径、连续 5 日 CVaR 窗口口径和历史压力恢复时间定义，并由黄金样例锁定；不得根据结果选择口径。
