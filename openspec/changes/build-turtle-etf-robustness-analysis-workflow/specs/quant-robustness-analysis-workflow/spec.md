## ADDED Requirements

### Requirement: 分析流程必须由显式配置驱动
系统 MUST（必须）接收调用者明确提供的正式结果清单和分析配置。清单 MUST 逐项声明运行身份、场景角色、归档路径、来源链接、代码与参数摘要、所需数据集及文件 SHA256（完整性摘要）；配置 MUST 声明指标口径、抽样参数、压力窗口、冲击映射、尾部定义和验收门槛。`analyze-quant-robustness` Skill（技能） MUST 只编排前置检查、脚本调用、停止状态和结果交付，不得包含具体策略的资产、参数、阈值或计算分支。

#### Scenario: 使用非海龟策略配置运行分析
- **WHEN** 调用者提供一个字段完整的非海龟策略正式结果清单及其项目分析配置
- **THEN** Skill（技能）调用同一套通用脚本完成分析，不要求存在 ETF（交易型开放式指数基金）、N 值、海龟参数或海龟项目文件

#### Scenario: 拒绝隐式选择正式结果
- **WHEN** 调用者未列出正式运行身份，或要求使用 `latest`（最新）结果、自动扫描替代快照或猜测缺失场景
- **THEN** 流程在读取分析数据前停止并输出“证据不足”，不得静默选择任何运行

### Requirement: 正式证据必须通过只读完整性门禁
系统 MUST（必须）在计算前验证每个归档对象的正式运行身份、完整性门禁、必需数据集、文件摘要、时间范围以及跨场景允许变化字段。分析过程 MUST 只读使用已经归档的正式证据，不得覆盖原始文件，不得补算缺失订单、成交、持仓或收益，也不得在本地创建新的正式运行身份。

#### Scenario: 完整正式证据通过门禁
- **WHEN** 清单中的全部对象均通过归档完整性门禁，文件摘要一致，且除配置明确允许变化的字段外代码、日期、费用和参数口径一致
- **THEN** 系统锁定本次输入摘要并进入确定性分析

#### Scenario: 必需正式数据缺失或冲突
- **WHEN** 任一必需数据集缺失、归档门禁失败、文件摘要不一致、运行身份冲突或跨场景出现未声明的口径差异
- **THEN** 系统列出受影响的要求和来源问题，将相应判定及整体结论标记为“证据不足”，且不尝试重建交易路径

### Requirement: 通用指标和区块抽样必须确定性可复现
通用脚本 MUST（必须）以规范化输入和显式配置计算 CAGR（复合年增长率）、最大回撤、Calmar（卡玛比率）以及基于连续收益区块的 Block Bootstrap（区块自助抽样）。抽样配置 MUST 包含区块长度、路径长度、路径数量和固定随机种子；实现 MUST 保留极端日、亏损日和亏损区块，并保存输入摘要、配置摘要、脚本版本和随机种子，使相同输入在相同运行环境中产生相同结果。

#### Scenario: 相同配置重复执行区块抽样
- **WHEN** 同一收益序列、区块长度、路径长度、路径数量、随机种子和脚本版本运行两次
- **THEN** 两次生成的路径统计、门槛指标和结果摘要完全一致

#### Scenario: 配置包含亏损和极端收益
- **WHEN** 输入收益序列包含极端亏损日、连续亏损区块和分位点边界值
- **THEN** 脚本不删除、缩尾、替换或重排这些观测，并把它们纳入指标和区块抽样

### Requirement: 历史压力和假设冲击必须使用正式路径证据
通用脚本 MUST（必须）支持配置化历史压力窗口和配置化假设冲击。历史窗口 MUST 从正式净值或收益路径计算并报告窗口回撤、最低点和恢复信息；假设冲击 MUST 对正式快照中的每日收盘后真实持仓及其可追溯风险字段做静态重估。冲击计算不得生成后续订单、成交、仓位演化或新的正式交易路径。

#### Scenario: 执行配置化持仓冲击
- **WHEN** 配置提供资产分类到冲击幅度的映射，正式快照提供某日收盘后真实持仓及所需风险字段
- **THEN** 脚本计算该日冲击后的账户损失，记录持仓来源和配置摘要，不推演冲击后的交易行为

#### Scenario: 冲击所需风险字段缺失
- **WHEN** 某项冲击需要共同止损、波动单位或资产分类，但正式快照没有对应字段或可验证映射
- **THEN** 该冲击及依赖它的结论标记为“证据不足”，不得使用默认值或从后续价格反推

### Requirement: CVaR 计算必须保留完整尾部样本
通用脚本 MUST（必须）支持配置化的非参数 Historical CVaR（历史条件风险价值）计算，将分位点相同的全部收益纳入尾部样本。多日收益 MUST 按连续实际收益复合计算，不得使用平方根时间近似；极端收益不得删除、缩尾、替换或修改。

#### Scenario: 分位点存在并列收益
- **WHEN** 尾部分位点上存在多条数值相同的收益记录
- **THEN** CVaR 计算把所有并列记录纳入尾部样本，并在输出中记录尾部样本数和分位点

#### Scenario: 计算配置化多日 CVaR
- **WHEN** 配置要求计算连续多日复合收益的 CVaR
- **THEN** 脚本使用实际连续复利构造多日收益，不使用单日波动率乘以时间平方根替代

### Requirement: 证据矩阵必须产生不可补偿的三态结论
系统 MUST（必须）为每项硬门槛记录稳定要求编号、正式证据来源、输入与配置摘要、计算值、比较符号、门槛和判定。单项判定 MUST 为“通过”“不通过”或“证据不足”；整体聚合 MUST 遵循以下顺序：存在必需证据缺失或不可验证项时整体为“证据不足”，证据完整但任一硬门槛失败时整体为“不通过”，仅全部硬门槛通过时整体为“通过”。已知失败 MUST 在整体“证据不足”时继续明确展示，任何指标不得补偿另一硬门槛失败。

#### Scenario: 证据完整但存在失败门槛
- **WHEN** 全部必需证据完整且至少一项硬门槛判定为“不通过”
- **THEN** 整体结论为“不通过”，即使其他收益、回撤或卡玛指标明显优于门槛

#### Scenario: 同时存在已知失败和缺失证据
- **WHEN** 某项硬门槛已经失败，同时另一项必需证据缺失
- **THEN** 整体结论为“证据不足”，证据矩阵仍保留并突出显示已知失败项，不得输出“通过”

### Requirement: 仓库 Skill 必须保持唯一实现和最小布局
仓库 MUST（必须）只在 `.agents/skills/analyze-quant-robustness/` 保存一份真实 Skill（技能）目录，包含精简的 `SKILL.md`、必要的 `scripts/`、按需读取的 `references/` 和 `agents/openai.yaml`（界面元数据）。`.claude/skills/analyze-quant-robustness` MUST 使用仓库相对 SymbolicLink（符号链接）指向该真实目录。业务数据、具体策略代码、项目专属配置、正式结果和分析证据 MUST NOT（不得）进入 Skill 目录，也不得创建第二套计算实现。

#### Scenario: Codex 和 Claude 发现同一 Skill
- **WHEN** 分别从 Codex（代码代理）和 Claude（代码代理）的仓库入口解析 `analyze-quant-robustness`
- **THEN** 两个入口指向同一真实目录，`SKILL.md` 和脚本文件摘要完全一致，且不存在复制目录

#### Scenario: Skill 结构快速校验
- **WHEN** 使用 Skill Creator（技能创建指南）的 `quick_validate.py` 检查真实 Skill 目录并运行仓库布局校验
- **THEN** 名称、YAML（配置格式）前言、触发描述、必要文件、相对符号链接和唯一实现约束全部通过

### Requirement: 验证必须覆盖发布入口和非海龟前向使用
实现完成前 MUST（必须）通过项目 `.venv` 运行确定性单元测试、配置契约测试和从真实 Skill 发布入口发起的 E2E（端到端）回归。E2E MUST 使用系统临时目录中的小型固定正式快照样例，覆盖输入门禁、指标、区块抽样、历史压力、配置化冲击、CVaR、证据矩阵及三态结论，且不得访问网络或历史业务归档。系统还 MUST 由独立子 Agent（子代理）使用一个非海龟最小策略样例执行前向验证；删除或隔离全部海龟项目文件后，通用校验和测试仍 MUST 通过。

#### Scenario: 从 Skill 发布入口执行 E2E 回归
- **WHEN** 调用者从仓库 Skill 入口启动 `self-test`（自检）
- **THEN** 同一生产编排和分析核心在临时目录完成全流程，验证通过、不通过和证据不足三类结果，且不访问聚宽或既有归档

#### Scenario: 独立 Agent 执行非海龟前向验证
- **WHEN** 独立子 Agent 只获得 Skill、非海龟正式快照样例和通用配置，不获得预期答案或海龟上下文
- **THEN** 该 Agent 能按 Skill 完成分析、解释停止状态和定位证据矩阵，且执行过程不读取任何海龟专属文件
