# Brainstorm Summary

- Change: build-turtle-etf-robustness-analysis-workflow
- Date: 2026-07-19

## 已确认的目标

- 用户将目标改为标准分析流程，覆盖深度归因分析与稳健性分析。
- 流程需要通用对接本地研究、聚宽回测与聚宽模拟交易，且保持只读分析边界。
- 现有 `analysis_data` 是本地研究和聚宽回测的共同读取入口；现有 `quant_analysis` 已实现部分归因、稳健性与证据矩阵能力。

## 确认的技术方案

- 显式来源登记逐项绑定 `scenario_id`、仓库相对路径、来源类型、预期清单摘要与模拟快照身份；拒绝 `latest`、目录扫描和隐式选择。
- 本地研究的四张共同事实表与聚宽回测、聚宽模拟交易的同名表字段已对齐；本地研究不落物理 `risk` 与 `period_risks`。聚宽回测和聚宽模拟交易都归档这两张物理表，模拟交易的根清单则指向指定快照下的数据文件。
- 本地研究、聚宽回测与聚宽模拟交易统一复用现有 `analysis_data` 只读视图；模拟交易仅在根清单的已验证数据路径与调用方指定快照一致时打开。当前读取器只接受聚宽回测，实施时以最小验证分支接纳模拟交易。模拟交易 `risk` 表可能带回测没有的官方字段；共同分析只使用已对齐的共同字段，并把额外字段作为来源参考保留。
- 共同分析始终从收益、权益、持仓、订单和独立基准计算；深度归因与每项稳健性检查按已验证证据能力运行，缺失时形成 `evidence_insufficient`（证据不足）而不补算或伪造。
- 本地归因使用结果包扩展，聚宽归因使用归档数据集；二者不共享物理声明，但只需要各自验证并投影为同一份只读归因输入，不能复制或改写归档。
- 交付保持现有确定性 JSON（结构化数据）与 Markdown（标记文档）报告，并增加来源能力、快照身份和未适用项的可追溯摘要。

## 关键取舍与风险

- 不创建新的统一结果包或按来源拆分的分析器；现有归档和本地结果包继续是唯一事实源。
- 活跃模拟交易可能在同步时变化；清单摘要、快照身份或数据路径任一不匹配即拒绝读取。
- 来源缺少归因或某项稳健性所需数据时必须保留“证据不足”，不得用推断或本地补算伪造证据。

## 测试策略

- 为三类来源各建立最小通过样例，并覆盖模拟快照身份、清单摘要和数据路径不一致时的拒绝。
- 覆盖有归因证据、归因缺失、单来源稳健性证据不足与多场景稳健性通过的结果。
- 从真实分析发布入口执行离线 E2E（端到端）回归；不访问网络、不启动上游流程，且验证原始归档摘要不变。

## Spec Patch

- 将当前海龟 ETF（交易型开放式指数基金）专属的提案、设计、任务和两份 delta spec（增量规格）改为标准分析流程；新增通用来源登记、三类来源只读验证、能力降级和可追溯交付验收场景。
- 删除“必须等待新聚宽回测流程完成”的硬依赖；分析只要求调用方提供通过完整性门禁的明确来源。
