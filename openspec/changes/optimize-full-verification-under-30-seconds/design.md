## Context

现有全量验证包含 11 个检查，检查级最多并行 7 路，多数 pytest（测试运行器）检查内部又启动 4 个 pytest-xdist（测试并行）工作进程。实测总耗时约 221–242 秒，其中本地量化单元测试约 96 秒、E2E（端到端）约 127 秒；主要瓶颈是重复进程初始化、嵌套过度并发和每个新进程重复执行 Numba JIT（即时编译）。

约束是保留全部测试、测试只在本机进程及其子进程内运行、不联网、不增加依赖，并且不修改上游 build-and-verify（构建与验证）运行时。

## Goals / Non-Goals

**Goals:**

- 让非缓存、连续的 `verify --full` 稳定不超过 30 秒。
- 保留全部单元测试、完整 E2E 回归和至少一个真实 JIT 执行校验。
- 仅修改仓库验证配置、直接相关的测试契约和等价性回归实际触发的仓库内计算热点。

**Non-Goals:**

- 不修改或复制上游 build-and-verify 运行时。
- 不用缓存命中、跳过测试或联网服务换取耗时。
- 不改变产品输出、正式聚宽运行边界或测试断言语义。

## Decisions

1. 以单层并行为主。删除检查内部的 pytest-xdist 工作进程，按独立测试域在检查级并行，避免检查级和测试级同时扩张进程。
2. 默认验证 Python 语义。非 JIT 专项测试设置 `NUMBA_DISABLE_JIT=1`，受限项目子进程仅透传这个 Numba 官方开关并直接执行同一被装饰函数；直接访问 `.py_func` 的测试使用 dispatcher 存在时的 Python 实现，否则使用函数本身。另在正常环境直接编译并执行一个现有生产 Numba 回调，证明真实 JIT 路径可用而不重复完整业务回归。
3. 合并同域入口但不合并共享状态。JoinQuant（聚宽）纯单元测试可共用 pytest 入口；本地量化单元测试、E2E 和真实 JIT 校验保持独立，使它们可并行且便于定位失败。
4. 以实测选择最小配置。比较候选工作进程数与检查分组，只保留能通过全部测试且连续全量运行均不超过 30 秒的最小配置。
5. 使用标准库 `os.walk` 的目录剪枝替代对每个文件重复执行 `Path.rglob`、路径解析和状态查询；外部写入守卫继续覆盖版本管理源码和归档，但排除已约定为并发运行态根目录的 `.local`。
6. 优化冻结等价性回归中的确定性计算。连续价格派生用 NumPy（数组计算）一次完成数值校验、基准变化识别和累计因子计算，只遍历少量公司行动日；现金二分遇到与已验证可行仓位完全相同的整数目标时复用结论，并在浮点中点不再变化时结束。两项优化均由公司行动测试和三场景冻结等价性夹具约束输出不变。

## Risks / Trade-offs

- [关闭 JIT 可能漏掉编译兼容性] → 保留正常环境下的最小真实 JIT 执行校验。
- [E2E 并行写入共享目录] → 每个用例使用唯一临时身份，写入仅限被守卫排除的 `.local` 运行态；版本管理源码和归档变化仍使执行失败。
- [机器负载造成偶发超时] → 使用连续非缓存全量运行验收，不以单次最佳值放行。
- [配置与契约测试漂移] → 同步更新现有 `test_skill_layout.py` 配置断言，不新增第二份配置事实源。
- [计算热点优化造成结果漂移] → 同时运行公司行动规则测试、关闭 JIT 的冻结等价性场景和真实 JIT 最小入口，要求摘要与冻结夹具完全一致。

## Migration Plan

先调整测试兼容性和配置契约，再切换验证分组；任一步失败均可恢复原配置。最终以严格 OpenSpec 校验和连续全量验证作为放行门禁。

## Open Questions

无。工作进程数由本机实测决定，不形成新的可配置框架。
