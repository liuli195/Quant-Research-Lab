## Context

整体架构固定为三个互相独立的 Skill（技能）：`run-local-quant-research` 负责本地研究流程，`run-joinquant-backtests` 负责聚宽回测流程，后续策略分析 Skill 负责统一分析本地和聚宽结果。标准分析数据契约以仓库现有聚宽回测归档为物理基准：聚宽现有目录和文件 0 改动直读，本地 vectorbt（向量化回测框架）结果向该口径适配。三者不互相导入实现。本变更主要交付第一项 Skill 和统一读取能力；策略分析 Skill 另立变更，本变更在本地流程之外用确定性分析验证统一数据包。Vibe-Trading（氛围量化）的群体分析为已知缺陷路径，禁止使用；没有可用单体分析入口时只记录证据不足。

本变更内部仍分为四层：仓库级 `run-local-quant-research` Skill 负责流程编排；`scripts/research/` 共用脚本负责共享日线行情中心、标准数据包、运行身份和证据；`strategy-003/research` 负责海龟 ETF（交易型开放式指数基金）项目适配、策略、参数和研究计算；`.local/` 保存不进入公开仓库的行情、运行证据和独立分析证据。当前仓库已有 `joinquant-archive-sync`（聚宽归档同步）、项目 `.venv`（虚拟环境）、仓库级 Skill 布局和验证约定，应复用这些能力，且不得修改 `strategy-001`、`strategy-002` 或把本地结果冒充 JoinQuant（聚宽）正式回测。

现有逐日模拟在真实行情上把绝大多数时间消耗在 A1（同日共享预算分配）可行性枚举和组合风险重复计算，完整运行连续触发 10、30 和 60 分钟硬超时。该结果说明主要性能问题位于执行内核，而不是 Parquet（列式文件）行情或 DuckDB（嵌入式分析数据库）查询。本次增量设计同时替换 `strategy-003` 本地模拟内核、把分析数据契约抽离到中立模块，并切断本地研究流程对归因、稳健性和报告算法的直接依赖。真实基线还暴露出公司行动记账缺口和上限判断错误：本次以当日原始行情事实派生连续总回报价格，修复研究收益的机械跳变；公司行动元数据只做核对并显式区分时点可知与事后核对，明确保留研究级近似边界，同时修复被动超限语义并删除不属于海龟系统的全部流动性交易规则。

## Goals / Non-Goals

**Goals:**

- 建立配置驱动、与具体策略解耦的本地研究编排入口。
- 沉淀全策略共享的不可变 Parquet（列式文件）日线行情批次、快照引用、内存 DuckDB（嵌入式分析数据库）查询校验、标准分析数据、运行清单和证据索引等确定性脚本。
- 以聚宽现有 `manifest.json` 和六类数据集建立标准分析读取契约；聚宽零转换直读。本地 vectorbt 生成共同必需的 `results`、`balances`、`positions`、`orders` 四类执行事实，官方 `risk` 与 `period_risks` 显式标记为来源未提供，避免本地研究流程预先计算策略分析结论。
- 建立真实 `strategy-003` 身份，完成海龟 ETF 共享行情快照接入、策略规则验证和单场景完整本地主流程；冻结基线和六个预设挑战由主 agent（代理）分别调用该 Skill 七次，再在独立分析目录聚合来源。
- 用策略自有、机器可读且版本化的 `analysis-plan.json` 定义基础场景和完整稳健性范围；通用分析只按 Schema（结构约束）校验和展开，Skill 不读取该计划。
- 使用 vectorbt（向量化回测框架）官方 `Portfolio.from_order_func()`（自定义订单函数）接管本地模拟、订单处理、共享现金、持仓、费用和权益记账，把海龟状态、A1 和风险规则迁入项目专属 Numba（即时编译）回调，并按已确认规则夹具验证新实现。
- 共享行情批次同时固化原始未复权行情与可核验公司行动；本地 vectorbt 在内存中只按生效日可见的 `上一交易日原始 close / 当日原始 pre_close` 派生连续总回报价格。公司行动元数据只做核对和审计：公告日在生效日后时标记为 `retrospective_reconciliation`（事后核对），绝不参与因子幅度或交易决策。信号、风险、成交与估值统一使用连续经济价格和经济单位；拆分和现金分红不得制造虚假损益，现金分红按除权日隐含再投资近似，不精确模拟支付日现金或真实份额。
- 海龟策略层不读取成交额，不设置最低成交额、单笔成交额占比、订单参与率或任何流动性门槛；流动性只属于进入策略前的 ETF 池筛选。
- 完成切换后删除旧自研逐日执行引擎、旧状态与交易对象、兼容导出、旧入口和只服务旧实现的测试，使 vectorbt 路径成为代码库中唯一的本地海龟执行方案。
- 本地研究流程完成后，在独立 `analysis_id` 下由通用确定性分析生成一份绑定兼容结果摘要的本地分析报告与推荐，证明统一读取入口可被真实下游消费；Vibe 仅允许无已知缺陷的单体公开能力，且不得成为报告完成条件或结论来源。
- 通过 Skill 初始化、结构校验、单元测试、用户入口 E2E（端到端）回归和非海龟前向验证证明通用性。

**Non-Goals:**

- 不在 Skill（技能）中保存海龟参数、资产池、交易规则、策略代码或研究证据。
- 不在本地运行或宣称正式回测、模拟交易和最终验收。
- 不重复实现聚宽认证、远端回测/模拟对象归档或修改 Vibe-Trading（AI 研究助理）上游源码。
- 不实施聚宽正式复核，不创建或修改策略分析 Skill，不新增或修改 Vibe-Trading 的职责约定、适配器或分析身份模型；本变更只记录现有公开能力的安全调用与边界证据，禁止群体分析。
- 不建设行情服务、持久数据库平台、远端数据仓库或通用交易插件；共享行情中心只是 `.local/` 文件式能力。
- 不把 vectorbt 封装成新的仓库通用回测引擎，不让 `scripts/research/`、行情中心、归因、稳健性或报告层依赖海龟回调，也不保留旧引擎、兼容层、双引擎开关或两个模拟内核并行裁决。
- 首版不实现分钟线、基本面、财务或因子数据，也不为以后扩展提前增加来源插件框架。

## Decisions

### 1. 使用“Skill 编排 + 共用脚本 + 项目适配器 + 本地证据”四层结构

`SKILL.md` 只描述执行顺序、输入输出、停止状态和安全边界。`scripts/research/local_quant_research/` 的 Python（编程语言）入口读取 JSON（结构化清单）配置，调用 `scripts/research/market_data/` 验证共享快照，再以参数数组安全调用仓库内项目入口并收口证据。`strategy-003/research` 只实现海龟项目适配和纯计算模块；行情与运行证据分别位于 `.local/market-data/` 和 `.local/quant-research/strategy-003/`。

替代方案是为海龟直接编写专用 Skill，或建立通用策略框架。前者无法复用且形成反向依赖，后者会提前抽象交易领域，因此均不采用。

本地研究 Skill 每次只接收一个场景并输出一份聚宽口径兼容结果，以 `next_action=return_to_caller` 停止；它拒绝候选数组，不循环七个方案，也不生成聚合清单。主 agent 先在 `preparation_id` 下生成七份配置并负责复数调用，随后显式登记七组 `scenario_id -> run_id`；分析入口校验同一快照、代码和执行后端后才派生独立 `analysis_id` 聚合来源，不扫描历史目录猜测结果，也不覆盖同计划的旧分析。聚宽回测 Skill 继续使用当前归档产物，不增加任何标准化步骤；策略分析 Skill 以后只通过统一读取入口消费现有聚宽目录或本地兼容目录。本地研究 Skill 不调用它。为完成本变更的实际研究交付，Codex 在 Skill 外部单独运行一次确定性策略分析；Vibe 仅可作为可选的安全单体解释器，不是本地流程节点，也不决定验收状态。

### 2. 共享行情中心使用不可变 Parquet 批次与快照引用

`.local/market-data/batches/<batch_id>/` 保存权威 `market-data.parquet`、`corporate-actions.parquet`、`manifest.json` 和 `validation.json`。前者保存原始未复权日线，后者保存版本化且可核验的拆分、现金分红等公司行动、公告日、应用日期、状态、知识截止日和来源摘要；二者共同构成本地行情事实，聚宽 CSV（逗号分隔文件）只在传输暂存阶段存在。派生连续总回报价格只在内存生成，不固化为第二套行情。`.local/market-data/snapshots/<snapshot_id>.json` 只引用一个或多个已验证批次及明确证券、日期、字段、来源、价格口径和公司行动摘要，不复制行情。相同内容按来源身份、结构版本和规范化逻辑内容摘要复用；Parquet 字节摘要单独验证文件完整性；新标的或新日期追加新批次；同一来源、频率、证券和日期出现不同值时拒绝合并，不覆盖旧批次。

DuckDB 只用 `:memory:` 和 `read_parquet` 建立可重建查询视图。通用脚本对字段顺序、类型、空值、排序和 `paused` 布尔类型规范化后比较清单逻辑摘要与查询结果摘要；不保存持久 `.duckdb` 副本。替代方案是每策略保存快照或以中央持久 DuckDB 为唯一事实源；前者重复且会分叉，后者需要额外时间旅行、迁移和损坏恢复，因此均不采用。

### 3. 海龟交易语义全部留在项目层

先创建真实聚宽策略空壳并同步为 `strategy-003`，只建立身份而不启动正式回测。海龟资产池、55 日入场、20 日 N 值、0.5N 加仓、2N 共同止损、20 日退出、风险上限、A1 分配及研究输出由项目配置和模块实现。通用入口只检查配置契约、调用项目入口并记录结果，不解释任何交易字段。海龟项目只引用共享 `snapshot_id`，删除海龟项目后 Skill、共享行情中心和非海龟 E2E（端到端）仍须通过。成交额可以作为共享行情字段供其他消费者使用，但不得进入海龟 `SimulationInputs`（模拟输入）、回调、配置或原因码；低成交额和成交额缺失都不能缩小、拒绝或延迟海龟订单。

替代方案是把常见指标或交易规则放入 Skill 脚本。即使名称通用，也会使流程层耦合当前策略，因此不采用。

### 4. 使用 vectorbt 官方自定义订单函数替换项目模拟内核

`strategy-003` 使用 vectorbt 官方 `Portfolio.from_order_func()`，不直接调用底层私有模拟函数，也不另建独立 Numba 引擎。11 只 ETF 组成一个 `cash_sharing=True`（共享现金）的组合组；预先对齐的连续经济 OHLC、经济单位、指标、协方差、公司行动审计信息和市场状态数组作为只读输入传入官方回调生命周期。原始未复权行情只在输入构造和审计层保留，不与连续经济价格混入同一账户：

1. `pre_sim_func_nb` 初始化批次、共同止损、候选、原因码和临时订单数组；
2. `pre_segment_func_nb` 在每个交易日开始时读取上一已完成交易日信号，分类退出、强制减仓和买入候选，并设置“退出 → 风险减仓 → 买入”的调用顺序；
3. `order_func_nb` 先执行卖单，进入第一个买单前基于卖单实际成交后的现金与持仓只计算一次 A1 和组合风险，再为各 ETF 返回最多一个订单；
4. `post_order_func_nb` 只在 vectorbt 返回实际成交后更新批次、理论档位和共同止损，拒单、停牌和涨跌停不虚构状态变化。

当前规则保证同一 ETF 同一交易日最多一笔最终订单，因此默认使用普通 `from_order_func()`，不启用 `flexible=True`（灵活多订单）。若未来规则确实需要同一 ETF 同一根 K 线多笔订单，必须另立规格变更。vectorbt 负责时间遍历、订单处理、共享现金、费用、滑点、持仓、交易和权益记录；项目回调只负责海龟决策、市场可交易性、A1、风险门槛和审计原因码，不负责流动性筛选或订单参与率控制。

每个交易日以当时可见的原始行情事实计算连续因子。当有效公司行动授权当日 `pre_close` 与上一交易日原始 `close` 的基准变化时，以 `上一交易日原始 close / 当日 pre_close` 更新只从当日起生效的累计连续因子，并用同一因子换算当日及以后 OHLC 与 `pre_close`；生效日停牌时应用日延后到首个复牌行情日，不得用未来元数据回写此前历史。公司行动元数据只核对事件身份、类型和状态：公告日在生效日之前或当日时标记 `point_in_time`（时点可知），晚于生效日时保留并标记 `retrospective_reconciliation`；取消状态按取消日期相对快照截止日重建，截止日后的取消仍按当时有效处理，当前状态已取消但缺少取消日期时停止导出；官方拆分比例或每份现金仅作审计，均不决定因子幅度或订单；有效事件未发生价格基准变化时只记录审计。vectorbt 的信号、N 值、协方差、风险、成交和估值统一使用连续经济价格与经济单位。拆分时经济单位保持稳定；现金分红在除权日进入连续总回报，等价于隐含再投资，不在支付日另增共享现金。取消事件不得授权变化；若事件状态未知、知识截止日无效，或价格基准变化没有对应有效事件，运行必须以 `evidence_insufficient`（证据不足）停止，不得按价格跳变阈值猜测。

单 ETF 和资产组资金上限采用“不得恶化”语义：被动升至上限以上不触发额外退出，也不冻结其他 ETF 的新增风险；只禁止继续增加同一超限 ETF 或超限资产组。退出、止损和风险降低订单始终允许。

所有 T 日收盘信号和滚动统计必须在进入回调前显式错位到 T+1 日执行行；协方差只能使用当时已经完成的样本。验收直接依据已确认交易规则、规格场景和固定小型合成夹具，不执行旧 `process_day` 完整路径，也不安排新旧双跑。新路径通过后删除旧 `execution.py`、旧状态/风险/A1/信号实现、直接耦合分析算法的项目 `reporting.py` 及其专用测试与导出；纯 `indicators.py` 只有被新输入路径实际使用时才保留，不建立兼容适配器。

替代方案包括继续优化 Python 枚举、使用 `Portfolio.from_orders()`（预生成订单）、`Portfolio.from_signals()`（信号模拟）或独立 Numba 引擎。前者继续维护自建循环；中间两者无法可靠表达实际卖出后再做组合级 A1；后者重复建设订单、现金和记账能力，因此均不采用。

### 5. 每次运行以不可变清单、唯一状态和完整输出收口

`run_id` 由共享快照、项目配置、规范化单场景配置、代码摘要和执行后端身份共同确定；不同场景不能复用同一运行身份。产物先写暂存位置，同一场景的冷启动和预热执行、180秒门槛与规范化结果摘要一致性通过后，先删除预热副本和可丢弃暂存并确认清理，再把清理结果写入 `performance.json`、生成最终清单并校验全部摘要，最后只原子固化一份已整理的权威结果。发布后不再依赖写入或清理。只有全部门禁通过才输出 `complete`；执行前身份或输入缺失输出 `evidence_insufficient`；既有证据不一致、执行异常、确定性冲突或临时文件无法确认清理输出 `failed`。同一完整身份可验证后复用，身份变化产生新运行，失败重试保留前次尝试。

每次 Skill 调用必须只在 `.local/quant-research/strategy-003/<run_id>/backtests/<local_backtest_id>/` 生成一个场景的本地 `manifest.json`、`code.py`、`params.json`、`params_versions/`、`performance.json`、`data/`、四类聚宽口径共同执行事实和海龟必需归因日志。本地清单把 `risk` 与 `period_risks` 标为 `required=false`、`status=missing_at_source`、`reason=computed_by_strategy_analysis`。单场景成功后以 `next_action=return_to_caller` 收口；Skill 不读取 `analysis-plan.json`，不生成 `candidate-strategies.json`、`local-research-manifest.json`、绩效归因、稳健性、报告、推荐或人工决定。主 agent 分别调用七次后只在独立分析目录生成 `source-results.json`。

### 6. 标准读取契约以现有聚宽归档为基准，并用确定性分析验证消费

2026-07-14 对仓库现有归档的只读核对显示：120 个通过门禁的回测清单拥有同一组数据集；125 份 `results.parquet` 与 `period_risks.parquet` 结构一致，其他核心表的差异仅为列顺序、`cancel_time` 的空类型/字符串表现和少量风险数值整数/浮点表现。因此唯一契约位于 `scripts/research/analysis_data/`，以聚宽现有 `manifest.json` 和六类核心 Parquet 为输入，按字段名建立 DuckDB（嵌入式分析数据库）内存视图。它不要求聚宽生成 `analysis-data-manifest.json`、八表副本或转换目录，也不修改源文件摘要。

聚宽读取路径是 Identity Adapter（恒等适配）：直接验证现有清单、门禁、数据集状态、文件摘要和空表证据后读取。`data/official-summary.csv` 继续作为页面口径交叉验证，不进入确定性核心计算；`attribution_log`、日志、性能剖析和 `raw/` 对通用来源保持可选扩展或归档证据。完整往返交易、权益、基准和事件等分析视图由六表在查询期派生，不要求聚宽侧新增物理表。

本地 vectorbt 适配器承担兼容成本，生成四类共同执行事实与同类数据集清单；海龟专属审计写入项目强制归因日志，固定字段、确定性 `event_id` 唯一主键和 `turtle-etf-attribution/2` 原因码，并记录公司行动应用，缺失或无法覆盖实际订单、风险变化或公司行动应用时阻止完成。它不生成聚宽官方风险摘要。证据权限从既有聚宽 `object.kind=backtest`、来源身份和 `gate.status=pass` 推导；本地权限从本地对象类型和运行身份推导，不给现有聚宽清单追加 `authority` 字段。

本地结果从 `backtests/<local_backtest_id>/` 向内尽量与聚宽现有目录一致；本地清单明确使用 `object.kind=local_backtest`、`source.kind=local_vectorbt`、`authority=local_research`。由于不存在真实聚宽远端证据，本地不伪造详情 URL、`research_response`、`collection_fence`、`official_summary`、`raw/` 或官方风险文件；这些是有意保留的最小差异。

本地清单不复用聚宽远端归档 Schema，而使用 `scripts/research/analysis_data/schemas/local-backtest-manifest.schema.json`，顶层版本固定为 `local-backtest/1`。读取器先看 `schema_version`：整数 `1` 只走现有聚宽 Schema；字符串 `local-backtest/1` 只走本地 Schema；未知版本、混合字段或失败后回退均拒绝。这样既保留聚宽零改动，也不要求本地伪造远端证据。

跨来源 Alpha/Beta 使用独立基准集 `.local/market-data/benchmark-sets/<benchmark_set_id>/`，只含沪深300人民币总回报和纳斯达克100人民币总回报，并记录币种、总回报/汇率口径、来源、日期、底层快照和摘要。只读实测确认聚宽 `results` 的 `returns` 与 `benchmark_returns` 为累计收益。为保持物理结构，本地 `results` 保留 `benchmark_returns:double nullable` 但全列为空，清单登记 `missing_at_source/independent_benchmark_set`；不得填零或选择一个双基准冒充。读取器按相邻累计净值比派生策略单日收益，并与双基准单日收益在共同有效交易日对齐，不得直接比较累计与单日序列。聚宽单基准也只作平台参考，不能同时冒充两个分析基准。

本地运行固化前先在 `.local/strategy-analysis-preparations/<preparation_id>/` 保存计划、基准、运行模板摘要和单场景配置；七份明确来源完成并通过共享身份校验后，由这些摘要派生 `.local/strategy-analysis/<analysis_id>/`。独立分析直接从权威 Parquet 通过统一读取器和内存 DuckDB 生成确定性数值、报告、推荐和 `analysis_seconds`。Vibe 的 `research-goal`（研究目标）和证据登记只能作为审计编排；加载 `performance-attribution`（绩效归因）、`risk-analysis`（风险分析）或 `report-generate`（报告生成）方法文档不等于完成分析。只允许调用无已知缺陷的单体分析入口；禁止 `run_swarm`（运行群体分析）、Vibe 回测和有前视偏差风险的组合优化器。若没有安全单体入口，`vibe-evidence.json` 记录 `evidence_insufficient`，报告仍由确定性分析完成。

`strategy-003/research/analysis-plan.json` 以 `strategy-analysis-plan/1` 机器可读定义一个冻结基线、六个挑战、固定时期、季度滚动、11个逐 ETF 删除、6个逐资产组删除、成本/延迟敏感性、区块抽样、历史压力、持仓冲击、CVaR、期望数量、固定种子和门槛。`quant_analysis` 只按通用 `analysis-plan.schema.json` 校验和展开封闭场景矩阵，不解析 Markdown、不导入海龟模块、不硬编码海龟常量。主 agent 只执行七次基础调用；其余场景从基线事实确定性计算，固定时期与滚动窗口是既有路径切片，资产删除不重新分配资金，成本/延迟是一阶订单级敏感性，报告必须声明这些限制。Vibe 不生成或补齐交易路径，任何群体结果均不得进入证据矩阵或报告结论。

替代方案是把分析直接塞进本地 Skill，或让本地与聚宽各自产生一套结果结构。前者破坏三 Skill 边界，后者会让策略分析维护两套适配，因此均不采用。

### 7. 沿用仓库运行环境与现有能力

所有本地 Python 命令使用 `.\.venv\Scripts\python.exe`。Skill 使用 `skill-creator`（技能创建指南）的 `init_skill.py` 初始化，以 `quick_validate.py` 校验，并按现有约定在 `.claude/skills/` 建立指向 `.agents/skills/` 的兼容符号链接。项目层固定 vectorbt 1.1.0 及其兼容的 NumPy（数组计算）、Pandas（数据处理）和 Numba 版本；安装或升级前先记录依赖差异，升级后运行仓库全量回归。通用流程在依赖缺失时只报告缺项，不静默安装，也不要求非海龟项目安装 vectorbt。

涉及现有聚宽认证和明确远端对象归档时调用 `joinquant-archive-sync`，不复制其浏览器、凭证或归档实现。共享聚宽日线导出器直接使用研究内核注入的 API（接口），兼容 Pandas 0.23.4 的 `line_terminator` 并规范化 `paused`；远端临时文件只有在本地字节摘要验证后才删除，无法确认清理时运行失败。

替代方案是为新 Skill 建独立虚拟环境或复制归档代码。两者会造成依赖和认证分叉，因此不采用。

### 8. 验证分为结构、确定性、性能、完整入口和通用性五层

验证依次覆盖 `quick_validate.py`、Skill 布局与符号链接、共享行情中心、通用运行器、单场景输入拒绝批量、现有聚宽回测目录零改动直读、双 Schema（结构约束）选择、六类逻辑视图、本地四类共同事实、全空 `benchmark_returns:double`、官方风险参考缺失声明、分析计划校验与展开、海龟必需归因日志、海龟规则夹具、官方 vectorbt 回调接线、性能、不变量和从 Skill 用户入口到一份单场景结果返回的完整海龟 E2E。主 agent 集成测试另行读取计划并调用七次，在分析目录聚合来源。每次单独回测从已准备输入进入 vectorbt 开始，到交易执行、四类共同事实、海龟必需归因日志及其结构/摘要/勾稽校验完成时停止；包含首次 JIT（即时编译）的冷启动回测和预热回测分别计时，两者都必须小于等于 180 秒且规范化结果摘要一致。停止计时后才写入并校验 `performance.json` 与最终清单，全部完成门禁通过后才发布一份权威结果。独立分析另记 `analysis_seconds`。

替代方案是只测试脚本函数。它不能证明 Skill 编排、项目适配和停止状态在真实入口上协同工作，因此不采用。

## Risks / Trade-offs

- [项目命令配置过宽可能执行非预期入口] → 只接受参数数组和仓库内明确路径，不经 Shell 拼接执行。
- [CSV 传输、Parquet 固化与 DuckDB 查询发生语义漂移] → 记录传输摘要，以固定结构转换，并分别校验逻辑内容与 Parquet 字节摘要。
- [权威快照字段或复权口径缺失] → 以 `evidence_insufficient` 停止，不用默认值补齐。
- [完整行情误入公开仓库] → 行情只写 `.local/market-data/`，最终验证扫描 Git（版本管理）跟踪文件。
- [批次重叠导致旧研究漂移] → 相同内容去重，冲突重叠拒绝，快照清单和旧批次不可修改。
- [持久 DuckDB 成为第二事实源] → 只使用内存视图并测试 `.local/market-data/` 不产生长期 `.duckdb` 文件。
- [通用契约被海龟需求逐步污染] → 通用测试使用非海龟夹具，并增加 Skill 目录不得包含海龟常量的边界检查。
- [Vibe-Trading 单体分析不可用或仍含已知缺陷] → 本地 `run_id` 不受污染；独立 `analysis_id` 记录证据不足，禁止转用群体分析，继续跳过受影响能力，不伪造 Vibe 报告。
- [统一契约反向要求聚宽改格式] → 以现有归档样例和 `manifest.schema.json` 为固定兼容门禁，聚宽目录只读；所有新增适配责任落在本地 writer（写入器）和分析 reader（读取器）。
- [本地身份无法通过聚宽远端 Schema] → 使用明确的 `local-backtest/1` Schema，并测试严格版本选择、混合字段拒绝和禁止回退。
- [单一聚宽 benchmark_returns 被误当成两个基准] → 双基准独立建集并记录总回报、汇率、来源和摘要；平台单基准只作参考。
- [七个基础方案或稳健性循环耦合进 Skill] → Skill 契约只接受一个场景；主 agent 复数调用并只在独立分析目录聚合。
- [分析场景只写在 Markdown 或硬编码进通用算法] → 策略自有版本化 `analysis-plan.json`，通用 Schema 只负责校验和展开，Skill 不读取计划。
- [稳健性敏感性被误读为独立路径回测] → 场景矩阵逐项记录来源与方法；只有七个基础场景来自独立回测，时期切片、资产删除和成本/延迟结果必须在报告中声明不重启交易路径、不重新分配资金或只作一阶估算。
- [vectorbt、NumPy 或 Pandas 升级破坏现有分析] → 固定经验证版本，升级前记录依赖差异并运行全部单元、E2E 和 Build and Verify（构建与验证）门禁。
- [vectorbt 的 Commons Clause 被误当成纯 Apache 2.0] → 在依赖清单记录官方许可和商业销售限制；本变更只用于仓库内部研究，不把 vectorbt 打包为对外销售的产品或服务。
- [回调顺序与共享现金产生语义漂移] → 单组共享现金、显式卖出优先调用顺序、卖单实际成交后一次性计算 A1，并以已确认规则和固定合成夹具逐项验证。
- [Numba 回调意外读取未来数据或使用 Python 对象] → 所有输入预先错位为只读数组，回调只接受稳定数值类型，并增加前视偏差和 `nopython`（无 Python 模式）编译测试。
- [公司行动缺失导致拆分或分红被记成亏损] → 批次必须同时包含可追溯公司行动；原始价、`pre_close`、事件和连续总回报因子分别验证，无法解释的除权差异关闭运行。
- [研究级近似被误读为真实账户] → 本地清单强制记录连续经济价格、经济单位、除权日隐含再投资、不支持支付日现金和不能精确对账聚宽；统一读取器与报告必须原样展示这些限制。
- [策略外流动性规则再次进入海龟执行] → `SimulationInputs`、回调和配置均不含成交额或参与率参数，并用低成交额、空成交额与正常成交额结果完全一致的测试和全仓符号扫描守住边界。
- [被动超限冻结全部买入] → 上限测试区分“已有被动超限”和“新增恶化”，只阻止同一证券或资产组继续增加，不新增强制退出。
- [删除旧实现后仍有隐藏引用或重复规则] → 在同一切换任务中删除旧模块、专用测试和公开导出，以导入测试、公开入口 E2E（端到端）和全仓符号扫描证明不存在兼容层、死代码或第二套规则实现。
- [JIT 编译时间掩盖真实性能] → 主 agent 的每次单场景调用都记录冷启动与预热耗时并直接应用 180 秒硬门槛；复数调用整体耗时和独立分析耗时只作运营信息，不替代单次门禁。
- [性能或清理证据晚于完成目录导致不合格结果被误用] → 冷/热执行和摘要一致性通过后，先清理预热副本及可丢弃暂存，再写 `performance.json` 与最终清单，最后原子发布；发布后不依赖写入或清理，失败只保留尝试证据。
- [独立本地稳健性分析被误读为正式通过] → 本地清单明确本地对象类型与运行身份，独立确定性报告声明非正式并在推荐后强制人工确认；聚宽官方权限只从既有归档身份和门禁推导。
- [完整 E2E 无法证明聚宽真实成交] → E2E 只证明本地规则与状态流；正式交易路径留给独立聚宽回测变更。
