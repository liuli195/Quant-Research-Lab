## Context

本变更同时交付两层能力：仓库级 `run-local-quant-research` Skill（技能）负责可复用的本地研究编排，海龟 ETF（交易型开放式指数基金）项目负责具体数据、策略、参数、研究计算和证据。当前仓库已有 `joinquant-archive-sync`（聚宽归档同步）、项目 `.venv`（虚拟环境）、仓库级 Skill 布局和验证约定，应复用这些能力，且不得修改 `strategy-001`、`strategy-002` 或把本地结果冒充 JoinQuant（聚宽）正式回测。

## Goals / Non-Goals

**Goals:**

- 建立配置驱动、与具体策略解耦的本地研究编排入口。
- 沉淀快照、CSV（逗号分隔文件）/DuckDB（嵌入式分析数据库）同源校验、运行清单和证据索引等确定性脚本。
- 完成海龟 ETF 权威数据快照接入、策略规则验证、完整本地主流程和探索性粗筛。
- 通过 Skill 初始化、结构校验、单元测试、用户入口 E2E（端到端）回归和非海龟前向验证证明通用性。

**Non-Goals:**

- 不在 Skill（技能）中保存海龟参数、资产池、交易规则、策略代码或研究证据。
- 不在本地运行或宣称正式回测、模拟交易和最终验收。
- 不重复实现聚宽认证、远端回测/模拟对象归档或修改 Vibe-Trading（AI 研究助理）上游源码。
- 不为扩展性预建服务、数据库平台或插件；除完成明确数据桥所必需外不新增依赖。

## Decisions

### 1. 使用“Skill 编排 + 单一确定性入口 + 项目适配器”三层结构

`SKILL.md` 只描述执行顺序、输入输出、停止状态和安全边界；单一 Python（编程语言）入口读取 JSON（结构化清单）配置，执行快照校验、数据桥校验、项目命令调用和证据汇总；项目适配器实现策略与研究逻辑。项目命令使用参数数组而非 Shell（命令解释器）字符串，并由配置显式给出仓库内路径。

替代方案是为海龟直接编写专用 Skill，或建立通用策略框架。前者无法复用且形成反向依赖，后者会提前抽象交易领域，因此均不采用。

### 2. 原始聚宽快照是唯一数据权威，CSV 与 DuckDB 是可重建视图

项目快照清单记录来源、快照标识、数据截止日、复权口径、字段、证券列表、导出代码摘要及文件 SHA256（文件摘要）。通用脚本对规范化后的行顺序、字段顺序、空值和类型计算内容摘要，CSV 与 DuckDB 必须指向同一快照且摘要一致；任一不一致即停止研究。

替代方案是把 CSV 或 DuckDB 单独作为权威。该方案无法证明两种研究入口同源，也会产生双重事实源，因此不采用。

### 3. 海龟交易语义全部留在项目层

海龟资产池、55 日入场、20 日 N 值、0.5N 加仓、2N 共同止损、20 日退出、风险上限及研究门槛由项目配置和模块实现。通用入口只检查配置契约、调用项目入口并记录结果，不解释任何交易字段。海龟项目删除后，Skill 的结构校验和通用测试仍须通过。

替代方案是把常见指标或交易规则放入 Skill 脚本。即使名称通用，也会使流程层耦合当前策略，因此不采用。

### 4. 每次运行以不可变清单和明确状态收口

运行身份由配置、代码和快照摘要共同确定，产物清单记录输入、命令、状态、输出路径和输出摘要。只有全部必需门禁通过才输出 `complete`；输入缺失或不可追溯输出 `evidence_insufficient`；执行或一致性校验失败输出 `failed`。已验证的同身份产物可复用，内容变化必须产生新身份，不覆盖既有完整证据。

替代方案是维护可变的 `latest`（最近一次）目录。该方案容易混淆不同代码和数据版本，因此不采用。

### 5. 沿用仓库运行环境与现有能力

所有本地 Python 命令使用 `.\.venv\Scripts\python.exe`。Skill 使用 `skill-creator`（技能创建指南）的 `init_skill.py` 初始化，以 `quick_validate.py` 校验，并按现有约定在 `.claude/skills/` 建立指向 `.agents/skills/` 的兼容符号链接。优先使用标准库和仓库已具备的 DuckDB、Pandas（数据处理）等依赖；只有现有环境无法完成明确需求时才增加并固定依赖。

涉及现有聚宽认证和明确远端对象归档时调用 `joinquant-archive-sync`，不复制其浏览器、凭证或归档实现；海龟行情导出代码属于项目产物，在聚宽研究环境生成明确快照后交给通用流程。

替代方案是为新 Skill 建独立虚拟环境或复制归档代码。两者会造成依赖和认证分叉，因此不采用。

### 6. 验证分为结构、确定性、完整入口和通用性四层

验证依次覆盖 `quick_validate.py`、Skill 布局与符号链接测试、通用脚本单元测试、从 Skill 用户入口到证据索引的海龟 E2E（端到端）回归。另以不含海龟词汇和资产的最小非海龟配置做独立前向验证；验证代理只获得 Skill 和原始任务/夹具，不获得预期答案。正式网络数据获取另行留证，不作为日常离线回归前提。

替代方案是只测试脚本函数。它不能证明 Skill 编排、项目适配和停止状态在真实入口上协同工作，因此不采用。

## Risks / Trade-offs

- [项目命令配置过宽可能执行非预期入口] → 只接受参数数组和仓库内明确路径，不经 Shell 拼接执行。
- [CSV 与 DuckDB 类型差异造成错误不一致] → 先按契约规范化类型、时区、空值和排序，再计算内容摘要。
- [权威快照字段或复权口径缺失] → 以 `evidence_insufficient` 停止，不用默认值补齐。
- [通用契约被海龟需求逐步污染] → 通用测试使用非海龟夹具，并增加 Skill 目录不得包含海龟常量的边界检查。
- [Vibe-Trading 已知前视偏差影响粗筛] → 官方修复前跳过受影响的组合优化器，其他探索结果仍明确标注为非正式。
- [完整 E2E 无法证明聚宽真实成交] → E2E 只证明本地规则与状态流；正式交易路径留给独立聚宽回测变更。

## Migration Plan

1. 使用项目 `.venv` 调用 `init_skill.py`，在 `.agents/skills/run-local-quant-research/` 初始化 Skill、脚本、参考和 `agents/openai.yaml`，再建立 `.claude/skills/` 兼容链接。
2. 实现通用 JSON 契约、确定性校验入口和离线夹具；接入仓库验证配置并运行 `quick_validate.py`、布局和单元测试。
3. 在项目目录新增海龟快照导出/接入、策略模块、配置、测试和研究报告，不向 Skill 目录复制业务内容。
4. 用固定夹具运行海龟完整本地主流程，再运行非海龟最小 E2E 和独立前向验证。
5. 若实施失败，删除本变更新增的 Skill、兼容链接、验证入口和海龟项目产物即可回滚；现有策略、归档格式和远端对象不迁移也不修改。

## Open Questions

- 新海龟策略的本地 `strategy_id`（策略标识）、聚宽名称和项目目录尚待深度设计确认。
- 权威行情快照的最终字段集合、复权方式和聚宽研究环境导出入口尚待用官方接口验证后确认。
- 同一开盘同时出现退出、风险减仓、建仓和加仓时的优先级，以及多订单争用预算和整手余数时的分配规则，尚待深度设计确认后写入海龟项目测试。
