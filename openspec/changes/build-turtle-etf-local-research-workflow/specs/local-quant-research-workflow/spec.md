## ADDED Requirements

### Requirement: 配置驱动的流程编排
系统 SHALL（必须）由仓库内明确的 JSON（结构化清单）配置接收快照、项目入口、输出和必需门禁；Skill（技能）只编排步骤，不得解释或硬编码具体策略规则。

#### Scenario: 有效配置运行项目研究入口
- **WHEN** 调用者提供通过契约校验的配置、明确快照和仓库内项目入口参数数组
- **THEN** 系统按“输入校验、数据桥校验、项目入口、证据汇总”的固定顺序运行，并记录每一步状态

#### Scenario: 项目输入不完整
- **WHEN** 配置缺少必需字段、使用 Shell（命令解释器）字符串或引用仓库外项目入口
- **THEN** 系统停止在项目代码执行前并输出 `evidence_insufficient`

### Requirement: 快照身份与完整性门禁
系统 SHALL（必须）验证来源、快照标识、数据截止日、复权口径、字段、证券清单、导出代码摘要和文件 SHA256（文件摘要），不得用隐式默认值补齐缺失身份。

#### Scenario: 快照清单完整且文件匹配
- **WHEN** 快照清单包含全部身份字段且重新计算的文件摘要与清单一致
- **THEN** 系统接受该快照并把身份和摘要写入本次运行清单

#### Scenario: 快照不可追溯或已被修改
- **WHEN** 任一身份字段缺失或任一文件摘要不匹配
- **THEN** 系统拒绝执行研究入口，保留错误证据并输出 `evidence_insufficient` 或 `failed`

### Requirement: 同源数据桥验证
系统 SHALL（必须）把 CSV（逗号分隔文件）和 DuckDB（嵌入式分析数据库）视为同一原始快照的可重建视图，并在规范化字段、类型、空值和排序后比较内容摘要。

#### Scenario: 两种入口同源一致
- **WHEN** CSV 与 DuckDB 声明相同快照标识且规范化内容摘要一致
- **THEN** 系统通过数据桥门禁并记录两种入口的文件摘要和共同内容摘要

#### Scenario: 身份或内容不一致
- **WHEN** 两种入口的快照标识、行集合、字段值或规范化内容摘要任一不一致
- **THEN** 系统在调用项目研究入口前失败，不选择其中一种入口继续

### Requirement: 可追溯且幂等的研究证据
系统 SHALL（必须）以配置、代码和快照摘要生成运行身份，并生成包含输入、命令、状态、输出路径和输出摘要的不可变证据索引。

#### Scenario: 相同身份重复运行
- **WHEN** 已存在同一运行身份且全部产物重新校验通过
- **THEN** 系统复用既有完整产物，不重写文件或创建第二份权威证据

#### Scenario: 输入或输出发生变化
- **WHEN** 配置、代码、快照或既有输出摘要发生变化
- **THEN** 系统不得覆盖原完整证据，并为新输入生成新运行身份或报告校验失败

#### Scenario: 完成状态门禁
- **WHEN** 所有必需步骤成功且必需输出存在并通过摘要校验
- **THEN** 系统仅此时输出 `complete`；输入证据不足输出 `evidence_insufficient`，执行或校验错误输出 `failed`

### Requirement: 仓库运行与能力复用边界
系统 MUST（必须）使用项目 `.venv`（虚拟环境）运行本地 Python（编程语言）入口，并复用既有聚宽认证和归档能力，不得保存或打印账号、密码、Token（访问令牌）或 Cookie（浏览器凭证）。

#### Scenario: 需要既有聚宽对象能力
- **WHEN** 流程需要认证或归档既有聚宽远端对象
- **THEN** Skill 调用 `joinquant-archive-sync`（聚宽归档同步）的公开入口，而不是复制其实现

#### Scenario: 运行环境或依赖缺失
- **WHEN** 项目 `.venv` 不存在或明确必需依赖不可用
- **THEN** 系统报告具体缺项并停止，不回退到系统 Python 或静默安装依赖

### Requirement: Skill 结构和通用性验证
实现 SHALL（必须）使用 `init_skill.py` 初始化 `run-local-quant-research`，通过 `quick_validate.py`、仓库布局测试、确定性脚本测试、用户入口 E2E（端到端）回归和非海龟前向验证。

#### Scenario: Skill 布局有效
- **WHEN** 运行结构与布局验证
- **THEN** `.agents/skills/run-local-quant-research/` 的元数据和必要资源有效，且 `.claude/skills/` 兼容链接解析到同一 Skill

#### Scenario: 通用能力与海龟项目解耦
- **WHEN** 在不提供海龟目录、参数、资产和代码的环境中运行通用单元测试及非海龟最小任务
- **THEN** Skill 仍能完成流程并生成可验证证据，且其目录不包含海龟专属常量或项目产物

#### Scenario: 用户入口完整回归
- **WHEN** 从 Skill 文档公开的用户入口启动离线研究夹具
- **THEN** 流程完整经过配置、快照、数据桥、项目入口和证据索引，而不是以若干孤立单元测试代替
