## 规格场景核验

- 专用 worktree（工作树）、运行锁和前置认证失败均在归档写入前处理。
- 同步结果、现有 `verify`（校验）和精确 Git（版本管理）路径共同构成提交门禁；非法结果失败关闭。
- 无变化不创建提交或 PR；有效变化只交给现有 PR Flow。
- 同步或校验失败执行受限回滚；回滚自身失败仍保留原始阶段、原因和恢复命令。
- PR Flow 启动或检查失败保留固定自动化分支，下一次只恢复发布，不启动新同步。
- 计划任务只改变动作参数；现有 `.venv`、工作目录、04:00、重试和单实例结构保持不变，生产任务未被替换。

## 验证范围说明

本地端到端回归使用临时 Git 远端和受控 JoinQuant（聚宽）/PR Flow 外部边界，没有访问真实 JoinQuant、创建真实 GitHub PR 或改动生产计划任务；这是规格明确要求的安全验证边界。真实远端发布由后续 PR 流程和合并后另行授权的生产任务迁移验证覆盖。
