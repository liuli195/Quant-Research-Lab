# standard-strategy-analysis-workflow Specification

## Purpose

定义量化分析如何以生产者和存储位置无关的方式，只读消费显式标准结果包、分析计划与独立基准，并产出可重复、可追溯的收益、风险、归因和稳健性结论。
## Requirements
### Requirement: 分析输入必须与结果包位置和生产者解耦

系统 MUST（必须）只要求调用方显式提供一个或多个标准结果包、一个分析计划和一个独立基准清单。结果包的生产者、父目录、文件名和传入路径 MUST NOT（不得）参与场景判断、计算分支或分析身份。读取器 MUST（必须）直接按标准结果包契约校验共同事实，不得经由生产者类型、执行引擎或遗留来源清单分支。

#### Scenario: 同一结果包被移动或复制

- **WHEN** 内容相同的结果包从不同路径传入
- **THEN** 系统产生相同的分析身份和结论

#### Scenario: 标准包不进入遗留来源分支

- **WHEN** 显式提供的结果包通过标准结果包契约校验
- **THEN** 系统使用该契约的固定读取路径验证并计算，不按生产者、执行引擎或遗留来源清单选择分支

#### Scenario: 输入未显式给出

- **WHEN** 调用方未提供结果包、分析计划或基准清单
- **THEN** 系统停止并报告缺失项，不扫描目录或猜测 `latest`（最新）对象

### Requirement: 结果包必须自证身份和完整性

系统 MUST（必须）从包内清单读取策略、场景和内容摘要，从冻结场景配置读取参数，并验证完成门禁和文件摘要。包内参数 MUST（必须）与分析计划中同一场景的展开参数完全一致；结果包的场景标识集合 MUST（必须）与分析计划的展开场景集合完全一致。重复场景、重复内容、身份不一致、缺少计划场景或额外场景 MUST（必须）被拒绝。

#### Scenario: 包内场景与计划不一致

- **WHEN** 结果包的冻结参数与分析计划不一致
- **THEN** 系统拒绝该结果包，不使用计划参数覆盖真实运行参数

#### Scenario: 计划场景缺少结果包

- **WHEN** 分析计划包含调用方未提供结果包的场景
- **THEN** 系统在生成交付前停止，并报告缺少和额外场景

### Requirement: 共同分析和扩展能力必须由数据契约驱动

系统 MUST（必须）使用 `results`（收益）、`balances`（资产）、`positions`（持仓）和 `orders`（订单）共同事实运行统一计算。扩展能力 MUST（必须）按声明字段识别；归因扩展的最小字段为 `time`（时间）、`event_id`（事件标识）和 `event_type`（事件类型），不得依赖扩展名称、执行引擎或生产者类型。

#### Scenario: 海龟归因使用自定义扩展名称

- **WHEN** 任意名称的完整扩展含归因最小字段且摘要、事件身份和时间范围有效
- **THEN** 系统将其作为归因证据，并原样保留已声明的详细字段

#### Scenario: 能力缺失

- **WHEN** 结果包缺少某项分析所需扩展
- **THEN** 系统将该项标记为 `evidence_insufficient`（证据不足），不阻断无依赖的共同分析

### Requirement: 来源原生亏损事件必须可审计

系统 MUST（必须）按来源原生 `valuation.security_daily_pnl`（估值事件单标的日盈亏）从小到大保留全部亏损事件，并在 Markdown（标记文档）报告中展示损失最大的前十条及事件总数。系统 MUST（必须）标识来源明确声明的退出事件，按亏损日期展示来源单标的盈亏合计、组合日盈亏和差额，并以 `0.02` 为勾稽容差。入场、事件前止损线、前一交易日信号、实际成交价或止损失败损失缺少语义匹配的来源字段时，系统 MUST（必须）逐字段标记 `evidence_insufficient`（证据不足），不得从活跃持仓或订单重建完整交易。

#### Scenario: 退出日没有活跃持仓行

- **WHEN** 来源归因扩展记录一笔退出日负 `security_daily_pnl`，而共同持仓事实没有该退出日证券行
- **THEN** 确定性 JSON（结构化数据）仍保留该亏损事件，Markdown 报告展示其标的、日期、盈亏、来源原因、退出标识和可用执行证据

#### Scenario: 可选执行证据来源缺失

- **WHEN** 亏损事件缺少入场、事件前止损线、前一交易日信号、实际成交价或语义匹配的止损失败损失
- **THEN** 报告逐字段显示证据不足，不从共同持仓或订单事实推断

### Requirement: 成本执行压力必须复用标准场景契约

佣金、单边滑点或额外交易日延迟压力 MUST（必须）作为分析计划中 `dimension=cost_execution`（成本执行）的普通场景声明。冻结场景配置 MUST（必须）是压力参数的唯一事实；分析不得维护第二份成本定义、从基线结果推导延迟成交或重新运行研究。

#### Scenario: 计算成本执行压力

- **WHEN** 调用方显式提供与分析计划完全匹配的成本执行场景结果包
- **THEN** 系统使用该结果包的真实收益、资产、持仓和订单事实计算 CAGR（复合年增长率）、最大回撤与 Calmar（卡玛比率），并按普通场景进入证据矩阵

#### Scenario: 成本执行场景缺失或参数冲突

- **WHEN** 计划声明的成本执行场景缺少结果包，或冻结配置与计划展开参数不一致
- **THEN** 系统按标准场景输入规则停止，不启动上游研究或在分析端补算

### Requirement: 硬门禁与诊断项必须分离

基线、参数场景和成本执行场景 MUST（必须）同时满足计划声明的 CAGR、最大回撤和 Calmar 门槛。滚动窗口硬门禁 MUST（必须）检查正 CAGR 窗口占比和任一窗口最大回撤，不增加 Calmar 门槛。固定时期、资产删除、Bootstrap（区块自助抽样）、历史压力、持仓冲击和 CVaR（条件风险价值） MUST（必须）作为诊断项交付，其失败或证据不足不得单独否决进入聚宽确认的建议。

#### Scenario: 诊断项失败但硬门禁通过

- **WHEN** 所有硬门禁通过，但一个或多个诊断项失败或证据不足
- **THEN** 系统保留完整诊断证据，并仍可建议进入聚宽确认

#### Scenario: 任一硬门禁失败

- **WHEN** 基线、参数场景、成本执行场景或滚动窗口硬门禁失败或证据不足
- **THEN** 系统建议修订后再进入聚宽确认

### Requirement: 分析交付必须确定、可追溯且只读

系统 MUST（必须）把结果包内容摘要、分析计划摘要、基准摘要、脚本版本和证据矩阵写入确定性 JSON（结构化数据）及 Markdown（标记文档）报告。分析前后输入身份不一致时 MUST（必须）停止。分析 MUST NOT（不得）修改结果包、启动上游流程、联网或读取凭证。

#### Scenario: 离线完整分析

- **WHEN** 调用者从唯一 Skill（技能）入口提供有效结果包、计划和基准
- **THEN** 系统离线完成分析与报告，且结果包字节保持不变
