## Context

当前仓库已有统一分析数据视图和确定性分析核心：本地研究结果包与聚宽回测归档均可读取，且收益、资产、持仓和订单字段已对齐。聚宽模拟交易也归档同名核心表，但其根清单指向指定快照下的数据，当前读取器尚未接纳该对象类型。归因证据在本地结果包扩展和聚宽归档数据集中的位置不同，不能假装物理格式相同。

本变更服务需要标准化复核已完成研究或聚宽结果的研究者和 Agent（代理）。本地分析只读取完整、可追溯的来源，不能补算成交、重建持仓或把本地结果声明为新的正式回测或模拟交易。

另一个关键约束是复用边界：`analyze-quant-robustness` Skill（技能）的入口脚本与量化分析计算包均自包含在其 `scripts/`（脚本）目录；它只通过中性的 `analysis_data`（分析数据）结果格式读取层消费标准结果包。海龟资产、参数、场景、压力定义和门槛必须由项目配置传入，并存放在 Skill 目录之外。

## Goals / Non-Goals

**Goals:**

- 建立与具体策略解耦、配置驱动的稳健性分析流程，接收明确的正式结果清单和分析配置。
- 以确定性脚本统一计算收益与回撤指标、Block Bootstrap（区块自助抽样）、历史压力、配置化假设冲击、CVaR（条件风险价值）和硬门槛证据矩阵。
- 保留输入文件摘要、计算配置、随机种子、脚本版本和输出之间的可追溯关系，使同一输入能够复现同一结果。
- 对明确登记的来源输出共同绩效、风险、独立基准、稳健性、可用深度归因和证据状态；不适用或缺失的项目明确输出“证据不足”。
- 通过结构校验、确定性测试、一次真实发布入口 E2E（端到端）回归和仓库完整验证，证明 Skill 能工作且未耦合海龟系统。

**Non-Goals:**

- 不在本地运行或重建正式交易路径，不补算缺失订单、成交或持仓。
- 不修改本地研究结果包、聚宽回测归档、聚宽模拟快照、策略规则或验收门槛，不按结果选择较优参数或删除不利场景。
- 不启动、提交、同步、恢复或修改本地研究、回测、模拟交易或任何上游流程。
- 不在 Skill（技能）目录保存海龟策略代码、业务数据、正式结果或项目专属配置。
- 不重复实现聚宽认证、回测运行或归档同步能力。

## Decisions

### 1. 采用“自包含 Skill + 显式来源登记 + 中性结果格式”三层结构

`.agents/skills/analyze-quant-robustness/SKILL.md` 只描述触发条件、前置检查、调用顺序、停止状态和结果交付；其 `scripts/quant_analysis/` 包保存实际计算。来源登记清单逐项列出场景、仓库相对路径、预期清单摘要、来源类型和模拟快照身份；读取只复用 `scripts/research/analysis_data/` 这个中性标准结果格式层，不保留仓库量化分析入口。

数据流固定为：

```text
显式来源登记 + 分析配置 + 独立基准集
→ 来源与快照完整性校验
→ 共同数据视图与能力识别
→ 确定性共同分析、归因和稳健性
→ 逐项证据矩阵与可追溯报告
```

选择该结构是因为 Skill 保存实际计算且只有一个公开入口，显式登记防止归档自动漂移，中性结果格式层则让生产与分析仅通过不可变结果衔接。

替代方案：复制三类来源到新格式，或为每类来源建独立分析器。两者会复制不可变证据或重复指标与报告，故不采用。

### 2. 只接受显式、不可变且已通过门禁的来源

输入加载器验证每个本地研究或聚宽来源的清单、门禁、文件摘要与场景身份。对于模拟交易，还必须验证调用方指定的快照身份等于根清单实际引用的数据快照，且所有核心数据路径位于该快照目录。任何不一致不得自动选择 `latest`（最新）、扫描替代快照或构造缺失交易数据。

选择显式清单可以防止归档或活跃模拟同步后分析对象静默漂移，并能把每一项结论绑定到唯一来源。

替代方案：按策略目录自动寻找最新完整回测。该方案省去清单维护，但无法证明使用的是预先确认对象，也会破坏重复执行，故不采用。

### 3. 共同分析与来源能力分离

共同分析只读取四张已对齐的事实表和独立基准集；聚宽官方风险表仅作为来源参考。归因加载器分别验证本地扩展和聚宽 `attribution_log`（归因日志），再投影为同一只读归因输入。每项稳健性检查声明所需来源能力，缺失时输出“证据不足”而非中止其他共同分析。

Block Bootstrap（区块自助抽样）使用连续收益区块和固定随机种子；Historical CVaR（历史条件风险价值）保留全部极端收益与并列分位点。历史压力和持仓冲击只使用来源已有的收益、持仓与可验证映射，不产生后续订单或新的交易路径。

选择纯计算接口便于用小型固定样例验证公式，并让非海龟策略复用同一实现。

替代方案：直接在报告脚本中串联读取、计算和渲染。该方案文件较少，但输入错误、数值错误和展示错误无法独立定位，故不采用。

### 4. 使用逐项三态证据矩阵形成结论

每个已请求分析项记录来源身份、能力状态、输入与配置摘要、计算值、比较符号、门槛和 `通过`、`不通过` 或 `证据不足`。共同分析、归因和稳健性分别聚合；缺失深度归因不否定已完成共同分析，但必须阻止任何依赖该归因的结论。

选择三态模型可区分“已经失败”和“还无法裁决”，避免把数据缺口误报为通过或失败。

替代方案：把缺失值统一当失败。该方案虽然保守，却掩盖需要补充证据而非修改策略的真实原因，故不采用。

### 5. 报告保持来源边界

报告保留来源类型、清单摘要、模拟快照身份、共同分析口径、归因能力和每项稳健性能力状态。本地研究近似口径不得描述为聚宽精确账户复核；聚宽官方风险的额外字段不得伪装为跨来源共同指标。

选择项目配置作为唯一专属入口，可以让方案审阅者直接复核海龟要求，同时保持 Skill 和脚本在删除海龟项目文件后仍能通过通用测试。

替代方案：在通用脚本中提供 `turtle-etf` 预设。该方案调用简短，但会形成隐式专用分支并使后续门槛变更污染共用能力，故不采用。

### 6. 发布入口和只读边界同时作为验收对象

验证包括确定性函数单元测试、来源登记和快照契约测试、Skill Creator 的 `quick_validate.py`（快速校验）、仓库布局校验、从 `.agents` 真实发布入口运行一次完整 E2E（端到端）回归以及仓库完整验证。E2E 使用临时目录和固定来源样例，覆盖三类来源、能力降级、证据矩阵和三态结论；不得访问网络、调用归档同步或修改原始文件。

选择多层验证是为了分别证明格式正确、数值正确、发布入口可用和能力真正通用。

替代方案：只运行单元测试。该方案无法发现 Skill 路由、路径布局和实际编排断裂，故不采用。

### 7. 本地研究与量化分析仅共享结果格式

`run-local-quant-research`（本地量化研究）只生成一个单场景标准结果包或执行只读晋升；它不读取分析计划、不会调用量化分析 Skill（技能）、不会生成推荐。`analyze-quant-robustness` 只接受已登记结果包，既不构造本地研究运行配置，也不调用本地研究 CLI（命令行接口）、运行器或晋升操作。

两者只共同依赖 `scripts/research/result_contract.py` 与 `scripts/research/result_package.py` 这一中立标准结果契约。`analysis_data`（分析数据）属于只读分析侧：本地研究不得导入它；量化分析也不得导入本地研究运行器。回归测试必须同时验证静态引用和真实导入图均不存在跨侧运行时依赖。

## Risks / Trade-offs

- [模拟交易在读取期间更新] → 验证清单摘要、指定快照和数据路径；任一变化即拒绝读取。
- [来源缺少归因或稳健性所需字段] → 保持共同分析，并把依赖项目标记为“证据不足”。
- [来源间口径不一致] → 以来源类型、核算精度和独立基准披露差异，拒绝未声明的跨场景比较。
- [静态假设冲击被误读为正式回测] → 报告明确其为只读本地重估，不生成订单、成交或新的聚宽运行身份。

## Migration Plan

1. 扩展现有来源验证、读取、归因与报告代码，不改动现有归档能力与历史快照。
2. 建立最小来源登记契约和三类固定样例，先通过单元测试和发布入口 E2E（端到端）回归。
3. 建立 Skill（技能）入口及 Claude（代码助手）发现链接，运行 `quick_validate.py`（快速校验）、布局校验和仓库完整验证；不保留旧量化分析入口兼容层。
4. 回滚时只移除新增分析入口、来源登记和报告产物，原始结果包与聚宽归档保持不变。

## Open Questions

- 调用方必须在分析前提供来源登记、门槛、随机种子、年化口径、CVaR（条件风险价值）窗口和压力定义；缺失时只能输出“证据不足”。
